Variables and Declarations

Three ways to declare variables:

  • const: Cannot be reassigned
  • let: Block-scoped, can be reassigned
  • var: Function-scoped, avoid in modern code

In React Native, prefer const by default, use let when reassignment is needed.

// Constants (cannot be reassigned)
const API_URL = 'https://api.example.com';
const MAX_ITEMS = 50;

// Variables that can change
let count = 0;
let isLoading = true;

// Avoid var in modern JavaScript
var oldWay = 'legacy'; // Not recommended

// Error: cannot reassign const
// API_URL = 'https://new-api.example.com';

Data Types

Primitive Types:

  • string: Text values
  • number: Integers and floats
  • boolean: true/false
  • null: Intentional absence of value
  • undefined: Unassigned value
  • symbol: Unique identifiers

Reference Types:

  • object: Collections of properties
  • array: Ordered collections
  • function: Callable objects
// Primitive types
const name = 'John Doe';        // string
const age = 30;                 // number
const isActive = true;          // boolean
const noValue = null;           // null
let undefinedVar;               // undefined
const uniqueId = Symbol('id');  // symbol

// Reference types
const user = {                  // object
  name: 'John',
  age: 30
};

const medications = [           // array
  'Aspirin', 'Ibuprofen', 'Tylenol'

Operators and Expressions

Common Operators:

  • Arithmetic: +, -, *, /, %
  • Assignment: =, +=, -=
  • Comparison: ==, ===, !=, !==
  • Logical: &&, ||, !
  • Ternary: condition ? expr1 : expr2

Always use === (strict equality) instead of == to avoid unexpected type coercion.

// Arithmetic operators
const sum = 5 + 10;          // 15
const product = 4 * 3;       // 12

// Assignment operators
let count = 0;
count += 5;                  // count = count + 5

// Comparison operators
const isEqual = 5 === '5';   // false (strict equality)
const isLooseEqual = 5 == '5'; // true (loose equality)

// Logical operators
const hasPermission = isAdmin && isActive;
const canAccess = isAdmin || hasSpecialAccess;

// Ternary operator
const status = isActive ? 'Active' : 'Inactive';

Control Flow: Conditionals

Conditional Statements:

  • if/else: Basic conditional branching
  • else if: Multiple conditions
  • switch: Multiple case evaluation
  • Ternary operator: Compact conditionals

In React Native, conditionals are often used for conditional rendering of components.

// if/else statement
if (user.role === 'admin') {
  console.log('Admin access granted');
} else if (user.role === 'editor') {
  console.log('Editor access granted');
} else {
  console.log('Limited access');

// switch statement
switch (medication.type) {
  case 'antibiotic':
    console.log('Take full course');
    break;
  case 'pain':
    console.log('Take as needed');
    break;
  default:
    console.log('Follow prescription');
}

// Ternary operator
const message = age >= 18

Control Flow: Loops

Loop Types:

  • for: Traditional counting loop
  • for...of: Iterate over iterable values
  • for...in: Iterate over object properties
  • while: Condition-based loop
  • do...while: Executes at least once

In React Native, array methods like map, filter, and reduce are often preferred over traditional loops.

// for loop
for (let i = 0; i < medications.length; i++) {
  console.log(medications[i]);
}

// for...of loop (arrays)
for (const med of medications) {
  console.log(med);
}

// for...in loop (objects)
for (const key in user) {
  console.log(`${key}: ${user[key]}`);
}

// while loop
let count = 0;
while (count < 5) {
  console.log(count);
  count++;
}

// do...while loop
let i = 0;
do {
  console.log(i);
  i++;
} while (i < 3);

Error Handling

JavaScript provides mechanisms to handle errors gracefully:

  • try/catch/finally: Handle exceptions
  • throw: Create custom errors
  • Error object: Built-in error type
  • Custom error types: Extend Error

Proper error handling is crucial in React Native to prevent app crashes and provide feedback to users.

// Basic try/catch
try {
  const data = JSON.parse(response);
  processData(data);
} catch (error) {
  console.error('Failed to parse data:', error.message);
} finally {
  // Always executes
  hideLoadingIndicator();
}

// Throwing custom errors
function calculateDosage(weight, age) {
  if (weight <= 0 || age <= 0) {
    throw new Error('Invalid patient data');
  }
  
  if (age < 18) {
    throw new Error('Pediatric dosing requires specialist');
  }
  
  return weight * 0.5;
}

// Using in React Native
try {
  const dosage = calculateDosage(patient.weight, patient.age);
  setDosage(dosage);
} catch (error) {
  setError(error.message);
  Alert.alert('Dosage Error', error.message);
}

Type Coercion

JavaScript automatically converts types in certain operations:

  • Implicit coercion: Automatic conversion
  • Explicit coercion: Manual conversion
  • Common coercion scenarios:
    • String + Number â†’ String
    • Boolean in conditionals
    • Loose equality (==)

Understanding type coercion helps prevent subtle bugs in React Native applications.

// Implicit coercion
const sum = '5' + 3;        // '53' (string)
const diff = '10' - 5;      // 5 (number)
const result = true + 1;    // 2 (number)

// In conditionals
if ('0') {                  // true (non-empty string)
  console.log('This runs');
}

if (0) {                    // false (zero)
  console.log('This does not run');
}

// Loose vs strict equality
console.log(5 == '5');      // true (coercion happens)
console.log(5 === '5');     // false (no coercion)

// Explicit coercion
const numStr = '42';
const num = Number(numStr);  // 42 (number)
const str = String(42);      // '42' (string)
const bool = Boolean(1);     // true (boolean)
