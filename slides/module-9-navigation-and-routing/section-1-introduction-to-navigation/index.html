Navigation Concepts

Navigation is a fundamental aspect of mobile applications that allows users to move between different screens and features:

Key Navigation Concepts

  • Screens/Routes: Individual views in your app
  • Navigation State: Current screen and history
  • Transitions: Animations between screens
  • Parameters: Data passed between screens
  • Deep Links: External links to specific screens

Navigation Challenges

  • Platform-specific navigation patterns
  • Complex navigation hierarchies
  • State persistence across app restarts
  • Handling back navigation
  • Authentication flows
Navigation in React Native

React Native doesn't include a built-in navigation solution. Instead, developers can choose from several third-party libraries, with Expo Router and React Navigation being the most popular options.

Navigation Patterns

Mobile applications use several common navigation patterns:

Stack Navigation

  • Screens are stacked on top of each other
  • Back button returns to previous screen
  • Common for hierarchical content
  • Example: Medication list → Medication details

Tab Navigation

  • Tabs at bottom or top of screen
  • Quick switching between main sections
  • Persistent across the app
  • Example: Home, Medications, Cart, Profile

Drawer Navigation

  • Side menu that slides in from edge
  • Good for many navigation options
  • Less frequently accessed features
  • Example: Settings, Help, About

Modal Navigation

  • Screen appears on top of current view
  • Often used for temporary tasks
  • Dismissible by swiping or tapping
  • Example: Add medication, Filter options

Navigation State Management

Navigation state represents the current screen and navigation history:

Navigation State Components

  • Routes: Array of screens in the stack
  • Index: Current screen index
  • Key: Unique identifier for each route
  • Params: Data passed to each route
  • Type: Navigation action type

State Management Challenges

  • Preserving state during navigation
  • Handling deep linking
  • Persisting navigation state
  • Synchronizing with app state
  • Type safety for route parameters
// Example navigation state
{
  "index": 1,
  "routes": [

      "key": "medication-details-1",

        "id": "med-123",

Navigation Lifecycle

Understanding the navigation lifecycle is crucial for managing component state and effects:

Screen Lifecycle Events

  • focus: Screen comes into view
  • blur: Screen goes out of view
  • beforeRemove: Screen is about to be removed
  • state: Navigation state changes

Common Lifecycle Use Cases

  • Fetching data when screen focuses
  • Pausing/resuming animations
  • Saving form data before navigating away
  • Confirming navigation with unsaved changes
  • Cleaning up resources when screen blurs
// Example of using navigation lifecycle events
import { useEffect } from 'react';
import { useFocusEffect } from '@react-navigation/native';

function MedicationScreen({ navigation }) {
  // Run when component mounts and on navigation focus
  useFocusEffect(
    React.useCallback(() => {
      // Fetch medication data
      fetchMedications();
      
      // Clean up when screen loses focus
      return () => {
        // Cancel any pending requests
        cancelPendingRequests();
      };
    }, [])
  );
  
  // Listen for navigation events
  useEffect(() => {
    const unsubscribe = navigation.addListener('beforeRemove', (e) => {
      // Prevent leaving with unsaved changes
      if (hasUnsavedChanges) {
        e.preventDefault();
        // Show confirmation dialog
        confirmNavigation(e.data.action);
      }
    });
    
    return unsubscribe;
  }, [navigation, hasUnsavedChanges]);
}

Deep Linking

Deep linking allows users to navigate directly to specific screens in your app from external sources:

Types of Deep Links

  • URL Scheme Links: Custom protocol (e.g., pharmeasy://)
  • Universal Links: Standard web URLs that open in your app (iOS)
  • App Links: Verified web URLs that open in your app (Android)
  • Deferred Deep Links: Links that work even if the app isn't installed

Deep Linking Use Cases

  • Sharing medication information
  • Email/SMS notifications
  • Push notification navigation
  • Marketing campaigns
  • QR code scanning
// Example deep link URLs
pharmeasy://medications/123                 // URL Scheme
https://pharmeasy.example.com/medications/123  // Universal/App Link

// Deep link configuration (simplified)
{
  "prefixes": [
    "pharmeasy://",
    "https://pharmeasy.example.com"
  ],
  "config": {
    "screens": {
      "Medications": {
        "path": "medications",
        "screens": {
          "MedicationDetails": {
            "path": ":id",
            "parse": {
              "id": (id) => id
            }
          }
        }
      }
    }
  }
}

Navigation Best Practices

Follow these best practices for a better user experience and maintainable code:

User Experience

  • Follow platform-specific patterns
  • Keep navigation consistent
  • Provide clear navigation cues
  • Optimize transitions for smoothness
  • Handle back navigation intuitively
  • Preserve state during navigation

Code Organization

  • Centralize navigation configuration
  • Type route parameters with TypeScript
  • Separate navigation logic from UI
  • Create reusable navigation components
  • Test navigation flows
  • Document navigation structure
TypeScript Integration

Use TypeScript to define route parameters for type safety and better developer experience:

// Type-safe route parameters
type RootStackParamList = {
  MedicationsList: undefined;
  MedicationDetails: { id: string; name: string };
  Checkout: { items: CartItem[] };
};

// Type-safe navigation hook
const navigation = useNavigation>();

// Type-safe navigation
navigation.navigate('MedicationDetails', { id: 'med-123', name: 'Ibuprofen' });

Navigation Libraries Comparison

React Native has several navigation libraries, with Expo Router and React Navigation being the most popular:

Feature Expo Router React Navigation
Approach File-based routing Component-based routing
Configuration Minimal (based on file structure) Explicit (navigator components)
Deep Linking Built-in (based on file structure) Requires configuration
TypeScript Support Good Excellent
Flexibility Medium (conventions-based) High (fully customizable)
Learning Curve Lower (familiar to Next.js users) Higher (more concepts to learn)
Expo Integration Native (part of Expo SDK) Good (works well with Expo)
Which One to Choose?

For new projects, especially those using Expo, Expo Router is recommended due to its simplicity and built-in deep linking support. For more complex navigation requirements or existing projects, React Navigation offers more flexibility and customization options.
