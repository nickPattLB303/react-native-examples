Why Navigation Matters

Navigation is one of the most critical aspects of user experience in mobile applications.

A well-designed navigation system:

  • Helps users understand where they are in the app
  • Makes it easy to move between different sections
  • Provides context and hierarchy for information
  • Supports users' mental models of how the app is organized
  • Enables efficient task completion
Under the Hood

Unlike web applications with a URL-based navigation system built into browsers, React Native requires explicit navigation libraries to handle screen transitions, history, and state.

Common Navigation Patterns

Stack Navigation

  • Screens stacked on top of each other
  • New screens pushed onto the stack
  • Going back pops screens off the stack
  • Perfect for sequential flows
  • Example: Medication list → Medication detail

Tab Navigation

  • Switch between main sections via tabs
  • Typically at bottom of screen (iOS/Android)
  • Parallel navigation between peers
  • Example: Home, Medications, Prescriptions, Profile
Emphasize that these patterns can be combined to create complex navigation structures. Stack navigation is the foundation of most mobile navigation patterns.

More Navigation Patterns

Drawer Navigation

  • Side menu pulled from edge of screen
  • For less frequently used navigation options
  • Can combine with tabs and stacks
  • Example: Settings, Help, Legal Information

Modal Navigation

  • Screens overlay the current screen
  • For focused tasks or quick actions
  • Typically dismissed by swiping down or tapping button
  • Example: Add medication, Quick edit, Filters

Stack Navigation Example


// Conceptual example of stack navigation
import { createStackNavigator } from '@react-navigation/stack';

const Stack = createStackNavigator();

function MedicationStack() {
  return (
    <Stack.Navigator>
      <Stack.Screen 
        name="MedicationList" 
        component={MedicationListScreen} 
        options={{ title: 'Medications' }}
      />
      <Stack.Screen 
        name="MedicationDetail" 
        component={MedicationDetailScreen}
        options={({ route }) => ({ title: route.params.name })}
      />
    </Stack.Navigator>
  );
}
Explain that the Stack.Navigator is a container that manages the navigation state and transitions between screens. Each Stack.Screen defines a route in the navigator.

Tab Navigation Example


// Conceptual example of tab navigation
import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
import { Ionicons } from '@expo/vector-icons';

const Tab = createBottomTabNavigator();

function MainTabs() {
  return (
    <Tab.Navigator>
      <Tab.Screen 
        name="Home" 
        component={HomeScreen}
        options={{
          tabBarIcon: ({ color, size }) => (
            <Ionicons name="home" color={color} size={size} />
          ),
        }}
      />
      <Tab.Screen 
        name="Medications" 
        component={MedicationsScreen}
        options={{
          tabBarIcon: ({ color, size }) => (
            <Ionicons name="medical" color={color} size={size} />
          ),
        }}
      />
    </Tab.Navigator>
  );
}

Navigation Libraries in React Native

React Navigation

  • Standalone library for React Native navigation
  • Component-based API
  • Extensive customization options
  • TypeScript support
  • Large community and ecosystem
  • Support for native navigation gestures

Expo Router

  • File-based routing library
  • Built on top of React Navigation
  • Web-like routing for React Native
  • Automatic route generation based on file structure
  • Built-in support for deep linking
  • Seamless integration with Expo
Under the Hood

Both libraries ultimately use native navigation components and bridge JavaScript to native platform code for transitions, gestures, and animations.

Choosing a Navigation Approach

Consideration React Navigation Expo Router
Project Structure Flexible, component-based File-based, more rigid but organized
Team Background Mobile developers Web developers (similar to Next.js)
Customization Fine-grained control Convention over configuration
Deep Linking Requires explicit setup Built-in, more straightforward
Project Size Works for any size Better for larger projects with many screens

The Navigation Stack

The navigation stack is a fundamental concept:

  • Works like a stack data structure (LIFO)
  • New screens are "pushed" onto the stack
  • Going back "pops" screens off the stack
  • The top screen in the stack is what the user sees
  • The stack maintains navigation history

Navigation Props

Screens receive navigation props that allow them to:

  • Navigate to other screens
  • Go back to previous screens
  • Access route parameters
  • Modify the navigation header
  • Listen to navigation events

function MedicationDetailScreen({ navigation, route }) {
  const { medicationId } = route.params;
  
  return (
    <View style={styles.container}>
      <Text>Medication ID: {medicationId}</Text>
      <Button 
        title="Go Back" 
        onPress={() => navigation.goBack()} 
      />
      <Button 
        title="Go to Instructions" 
        onPress={() => navigation.navigate('Instructions', { medicationId })} 
      />
    </View>
  );
}

Best Practices for Navigation

Structure & Design

  • Keep navigation structure simple and intuitive
  • Limit navigation depth (avoid deep nesting)
  • Provide clear visual indicators of current location
  • Use consistent navigation patterns throughout

Development & Maintenance

  • Implement proper type safety with TypeScript
  • Test navigation thoroughly on different devices
  • Consider accessibility in navigation design
  • Structure navigation code to be maintainable
Common Pitfall

Overly complex navigation structures with deep nesting can confuse users and make your app difficult to navigate. Keep it simple!
