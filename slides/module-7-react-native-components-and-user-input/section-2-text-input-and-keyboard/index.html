The TextInput Component

The TextInput component is the primary way to capture text input in React Native.

Key Characteristics

  • Basic text entry field for users
  • Highly customizable appearance and behavior
  • Platform-specific styling by default
  • Various keyboard types available
  • Can be controlled or uncontrolled

Platform Implementations

  • iOS: Maps to UITextField
  • Android: Maps to EditText
  • Web: Maps to input or textarea
Core Concept

TextInput is always a controlled component in best practice React Native development - you track its state in JavaScript.

For Web Developers

Note that in React Native, we use onChangeText which directly gives the string value, rather than onChange which would return an event object in web React.

Essential TextInput Props

Value and Changes

  • value: Current text value
  • defaultValue: Initial value (uncontrolled)
  • onChangeText: Called when text changes
  • onSubmitEditing: Called when submitting
  • onEndEditing: Called when editing ends

Appearance

  • placeholder: Hint text when empty
  • placeholderTextColor: Color of placeholder
  • secureTextEntry: For passwords
  • style: Styling for the input
  • selectionColor: Color of selection/cursor

Behavior

  • autoCapitalize: Text capitalization
  • autoCorrect: Enable/disable autocorrect
  • autoComplete: Autofill behavior
  • maxLength: Character limit
  • multiline: Enable multiple lines

Keyboard

  • keyboardType: Type of keyboard to display
  • returnKeyType: Label for submit key
  • blurOnSubmit: Dismiss keyboard on submit
  • onKeyPress: Called when key is pressed
  • autoFocus: Focus on mount

Keyboard Types

React Native provides different keyboard types optimized for various input scenarios:

Common Keyboard Types

  • 'default': Standard keyboard
  • 'number-pad': Only numbers
  • 'decimal-pad': Numbers with decimal
  • 'numeric': General numeric input
  • 'email-address': Optimized for email
  • 'phone-pad': For phone numbers

Best Use Cases

  • Medication Name: 'default'
  • Dosage Amount: 'decimal-pad'
  • Age: 'number-pad'
  • Prescription ID: 'numeric'
  • Patient Email: 'email-address'
  • Patient Phone: 'phone-pad'
Platform Differences

Keyboard appearance may vary between iOS and Android. Some keyboard types are platform-specific.

Keyboard Management

Managing the keyboard is essential for a good user experience in mobile apps.

Keyboard Challenges

  • Keyboard takes up significant screen space
  • Can hide important UI elements including inputs
  • Keyboard behavior varies by platform
  • Users expect intuitive input flow between fields
  • Default behavior often needs customization

React Native Solutions

  • Keyboard API for programmatic control
  • KeyboardAvoidingView for adaptive layouts
  • ScrollView to enable scrolling to inputs
  • Input focus management with refs
  • Input accessories and customization

Using the Keyboard API


import React, { useState, useEffect } from 'react';

      }
    );

    // Clean up listeners

      keyboardDidShowListener.remove();
      keyboardDidHideListener.remove();

Focus Management

Controlling focus between multiple input fields improves user experience.

Focus Management Techniques

  • Use refs to reference TextInput components
  • Call focus() and blur() methods
  • Handle onSubmitEditing to move to next field
  • Set returnKeyType for clear user guidance
  • Use blurOnSubmit={false} to keep keyboard

Return Key Types

  • 'done': When input is complete
  • 'next': Move to next field
  • 'search': For search inputs
  • 'send': For messaging
  • 'go': For navigation actions
UX Best Practice

Set appropriate returnKeyType values to guide users on what happens when they press the return key.

Platform Differences in Text Input

Feature iOS Android
Keyboard appearance Generally consistent Varies by device manufacturer
Autocorrect behavior More aggressive by default Less intrusive by default
Clear button Available with clearButtonMode Not available natively
Multiline input May need explicit height Text aligns to top with textAlignVertical
Selection handling Context menu with copy/paste Floating toolbar with options
Return key behavior Generally dismisses keyboard May add newline in multiline inputs
Universal Approach

Test your input components on both platforms and provide platform-specific adjustments using Platform.select when necessary.
