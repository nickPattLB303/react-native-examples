Selection Components Overview

Selection components allow users to choose from predefined options rather than entering free-form text.

Types of Selection Components

  • Picker: Dropdown-style selection from a list
  • Switch: Toggle between two states (on/off)
  • CheckBox: Select multiple options (community component)
  • RadioButton: Select one from several options (community component)
  • Slider: Select a value from a continuous range

Selection Component Benefits

  • Prevent input errors by limiting choices
  • Improve UX by showing available options
  • Reduce cognitive load for users
  • Enable faster input for common selections
  • Support accessibility through clear choices
Core Concept

Selection components should always be used when input options are from a finite, predefined set of values.

The Picker Component

The Picker component provides a dropdown-style selection interface.

Key Characteristics

  • Native dropdown on each platform
  • Select one value from multiple options
  • Uses platform-specific UI conventions
  • Different appearance on iOS vs Android
  • Requires community package installation

Platform Implementation

  • iOS: Spinning wheel picker UI
  • Android: Dropdown menu
  • Web: Similar to <select> element
Important Note

As of React Native 0.70+, Picker has been moved to a separate library. Install it with:
npm install @react-native-picker/picker

The Switch Component

The Switch component is used for binary (on/off) selections.

Key Characteristics

  • Toggle between two states (true/false)
  • Native appearance on each platform
  • Simple API with value and onValueChange
  • Can be customized with colors
  • No manual styling of shape/size

Platform Implementation

  • iOS: Maps to UISwitch
  • Android: Maps to SwitchCompat
  • Web: No direct equivalent (custom styled checkbox)
Usage Guidance

Switches are best for toggles that take immediate effect, while checkboxes are better for selections that apply after form submission.

Implementing CheckBox Components

React Native doesn't include a built-in CheckBox component, but we can use community libraries or create our own.

Community Libraries

  • @react-native-community/checkbox
  • react-native-paper's Checkbox
  • react-native-elements's CheckBox

Custom Implementation

  • Use Pressable with icon/image
  • Toggle state on press
  • Show different icons for checked/unchecked
  • Implement accessibility properties
Decision Point

Consider whether to use a community library or implement your own checkbox based on your app's design system and requirements.

The Slider Component

The Slider component allows users to select a value from a continuous range.

Key Characteristics

  • Visual representation of a range
  • Continuous or stepped value selection
  • Requires community package
  • Customizable track and thumb
  • Supports accessibility

When to Use Sliders

  • Selecting from a continuous range
  • Visual representation is important
  • Value precision can be approximate
  • Ranges with minimum/maximum values
  • Settings that benefit from visual position
Important Note

Slider has been moved to a separate community package. Install it with:
npm install @react-native-community/slider

Combining Selection Components

In real applications, you'll often combine multiple selection components in forms.

Component Selection Guidelines

  • Use Switch for boolean options
  • Use Picker for single selection from many options
  • Use Checkbox for multiple selections
  • Use Radio Buttons for mutually exclusive options (few choices)
  • Use Slider for ranges and visual values

Practical Examples

  • Medication Form: Radio buttons for route, picker for frequency
  • Patient Preferences: Switches for notification settings
  • Allergy Information: Checkboxes for selecting multiple allergies
  • Pain Assessment: Slider for pain level
  • Provider Selection: Radio buttons or picker depending on number

Platform Differences in Selection Components

Component iOS Android
Picker Wheel picker, typically shown as modal Dropdown menu, opens in place
Switch Sliding toggle with clear on/off positions Material Design toggle with color change
Slider Rounded thumb, subtle appearance Material Design thumb, more pronounced
Radio Button Not native, requires custom implementation Not native, requires custom implementation
Checkbox Not native, requires custom implementation Not native, requires custom implementation
Design Strategy

For native feel, use platform-specific patterns. For consistent cross-platform UI, use custom components or a UI library like React Native Paper.
