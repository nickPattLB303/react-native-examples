Why Accessibility Matters

  • Legal Requirements: Many countries require digital accessibility compliance
  • Broader User Base: 15% of the world population lives with some form of disability
  • Better User Experience: Accessible apps are often more usable for everyone
  • Ethical Responsibility: Ensuring everyone can use your healthcare applications

React Native Accessibility Properties

Property Description
accessible Indicates if the element should be treated as accessible (default: true)
accessibilityLabel Text read by screen readers when the element is focused
accessibilityHint Describes the action that will occur when interacting with the element
accessibilityRole Communicates the purpose of the element (button, link, header, etc.)
accessibilityState Describes the current state of the element (disabled, selected, checked, etc.)

Creating Accessible Custom Form Components

const AccessibleSegmentedControl = ({ options, onChange, defaultValue }) => {
  const [selectedValue, setSelectedValue] = useState(defaultValue || options[0].value);
  
  return (
    
      {options.map((option) => {
        const isSelected = selectedValue === option.value;
        
        return (
           {
              setSelectedValue(option.value);
              if (onChange) onChange(option.value);
            }}
            accessibilityRole="radio"
            accessibilityState={{ checked: isSelected }}
            accessibilityLabel={`${option.label}, ${isSelected ? 'selected' : 'not selected'}`}
          >
            
              {option.label}
            
          
        );
      })}
    
  );
};

Platform-Specific Accessibility Considerations

iOS

  • VoiceOver is the built-in screen reader
  • Supports dynamic type for font scaling
  • Has specific gestures for navigation
  • Test with: Settings > Accessibility > VoiceOver

Android

  • TalkBack is the built-in screen reader
  • Supports font scaling through system settings
  • Has different navigation gestures than iOS
  • Test with: Settings > Accessibility > TalkBack

Important: Test your forms with both VoiceOver and TalkBack whenever possible!

Testing Form Accessibility

Manual Testing

  1. Enable screen readers (VoiceOver/TalkBack)
  2. Navigate using screen reader gestures:
    • Swipe to move between elements
    • Double-tap to activate
    • Three-finger swipe to scroll
  3. Check reading order and element descriptions
  4. Verify error announcements

Automated Testing Tools

  • iOS: Xcode Accessibility Inspector
  • Android: Accessibility Scanner
  • Jest with accessibility-specific tests
  • Detox for E2E testing with accessibility

Accessible Form Best Practices

  1. Provide Clear Labels: Every input needs a descriptive label
  2. Use Proper Input Types: Match the keyboard to the data type
  3. Group Related Elements: Organize inputs logically
  4. Sufficient Contrast: Text must be readable
  5. Large Touch Targets: At least 44×44 points
  1. Descriptive Button Text: Avoid vague labels like "Click Here"
  2. Accessible Error Feedback: Announce errors to screen readers
  3. Keyboard Navigation: Ensure proper tab order
  4. Test with Screen Readers: Use VoiceOver and TalkBack
  5. Support Font Scaling: Allow text to resize with settings
