Core UI Components in React Native

Module 7 | 45-60 minutes

  • Compare React Native components to their web equivalents

Time: 45-60 minutes

Prerequisites: Basic React components from Module 6

Core UI Components Overview

Unlike the web where we have HTML elements, React Native provides a set of built-in components that map to native UI elements on each platform.

  • React Native components render to native UI elements
  • Components have consistent cross-platform APIs
  • Styling is handled through JavaScript objects, not CSS
  • Components may have platform-specific behavior or appearance
Key Insight

React Native's component-based approach ensures your app feels native on each platform while providing a consistent API for developers.

Emphasize that while the API is consistent across platforms, the actual rendering utilizes native UI components, which is why a React Native app looks and feels like a native app on each platform.

The View Component

The View component is the most fundamental building block in React Native.

Key Characteristics

  • Similar to <div> in web
  • Used for layout containers
  • Can be nested to create complex UIs
  • Supports flexbox layout
  • No direct styling inheritance

Platform Implementation

  • iOS: Maps to UIView
  • Android: Maps to ViewGroup
  • Web: Maps to div

View Component Example


import { View } from 'react-native';

const MedicationContainer = () => {
  return (
    
      
        {/* Child components go here */}
      
      
        {/* Footer content goes here */}
      
    
  );
};
Platform-Specific Note

Notice how shadows are implemented differently for iOS (shadowProperties) vs. Android (elevation).

The Text Component

The Text component is used to display text in React Native.

Key Characteristics

  • All text must be inside a Text component
  • Can be nested for styled text spans
  • Text styles can be inherited when nested
  • Can handle user interactions (press events)
  • Automatically adjusts for platform typography

Platform Implementation

  • iOS: Maps to UILabel or NSAttributedString
  • Android: Maps to TextView
  • Web: Maps to span or p
For Web Developers

Unlike HTML where text can be placed directly in many elements, in React Native all text must be wrapped in a Text component.

Text Component Example


import { Text } from 'react-native';

const MedicationDetails = ({ name, dosage, instructions }) => {
  return (
    <>
      
        {name} {dosage}
      
      
        Instructions: 
        {instructions}
      
    
  );
};
Nested Text

Note how Text components can be nested to apply different styles to parts of the text, similar to using <span> in HTML.

The Image Component

The Image component is used to display images in React Native.

Key Characteristics

  • Can display images from multiple sources
  • Requires explicit dimensions
  • Supports different resize modes
  • Can display loading indicators
  • Handles image caching automatically

Image Sources

  • Local static images: require('./image.png')
  • Network images: {uri: 'https://...'}
  • Base64 images: {uri: 'data:image/png;base64,...'}
  • Local files: {uri: 'file://...'}

Image Component Example


import { Image, View, StyleSheet } from 'react-native';

const MedicationImage = ({ uri, localImage }) => {
  // Different ways to load images
  return (
    
      {/* Network image */}
       console.log('Image loaded')}
      />
      
      {/* Local static image */}
      
    
  );
};

const styles = StyleSheet.create({
  container: {
    flexDirection: 'row',
    justifyContent: 'space-between',
  },
  networkImage: {
    width: 100,
    height: 100,
    borderRadius: 8,
  },
  localImage: {
    width: 80,
    height: 80,
    borderRadius: 40,
  },
});
Important

Always specify width and height for images. React Native cannot automatically determine image dimensions for network images like the web can.

The ScrollView Component

The ScrollView component provides a scrollable container for multiple components and views.

Key Characteristics

  • Generic scrollable container
  • Renders all children at once
  • Supports vertical and horizontal scrolling
  • Can handle keyboard dismissal
  • Supports "pull to refresh" functionality

When to Use

  • For content with fixed or small number of items
  • When all elements need to be rendered immediately
  • For forms that may exceed screen height
  • For complex layouts that need scrolling
  • For horizontal carousels or pagers
Performance Note

For long lists, use FlatList instead of ScrollView to avoid performance issues.

The FlatList Component

The FlatList component is a high-performance way to render scrollable lists.

Key Characteristics

  • Only renders items currently visible
  • Recycles list items for better performance
  • Built-in pull to refresh and load more
  • Supports horizontal or vertical orientation
  • Handles large data sets efficiently

Key Props

  • data: Array of items to render
  • renderItem: Function to render each item
  • keyExtractor: Function to extract unique key
  • onRefresh/refreshing: Pull to refresh
  • onEndReached: Load more when scrolled to end

The Pressable Component

The Pressable component is used to create interactive elements that respond to touch.

Key Characteristics

  • Detects press interactions on any component
  • Can respond to various press states
  • More flexible than older Touchable components
  • Can customize feedback appearance
  • Works on both Android and iOS

Interaction States

  • onPressIn: Touch begins
  • onPressOut: Touch ends
  • onPress: Successful press
  • onLongPress: Press held for a duration
  • pressed: Boolean state for styling

Pressable Component Example


import { Pressable, Text, StyleSheet } from 'react-native';

const MedicationButton = ({ onPress, title, danger }) => {
  return (
     [
        styles.button,
        danger && styles.dangerButton,
        pressed && styles.buttonPressed
      ]}
      android_ripple={{ color: 'rgba(0, 0, 0, 0.1)' }}
    >
      {({ pressed }) => (
        
          {title}
        
      )}
    
  );
};

const styles = StyleSheet.create({
  button: {
    backgroundColor: '#007AFF',
    paddingVertical: 12,
    paddingHorizontal: 20,
    borderRadius: 8,
    alignItems: 'center',
  },
  buttonPressed: {
    backgroundColor: '#0056B3',
    opacity: 0.9,
  },
  buttonText: {
    color: 'white',
    fontWeight: '600',
    fontSize: 16,
  },
  buttonTextPressed: {
    color: 'rgba(255, 255, 255, 0.8)',
  },
  dangerButton: {
    backgroundColor: '#FF3B30',
  },
  dangerText: {
    color: 'white',
  },
});
Android Specific

Notice the android_ripple prop that provides the material design ripple effect on Android.

Platform-Specific Components

React Native provides ways to handle platform-specific code and components:

Platform Module


import { Platform } from 'react-native';

// Conditional code
if (Platform.OS === 'ios') {
  // iOS-specific code
} else {
  // Android-specific code
}

// Platform.select
const styles = {
  container: {
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.2,
        shadowRadius: 4,
      },
      android: {
        elevation: 4,
      },
    }),
  },
};

Platform-Specific Extensions

  • Create separate files with .ios.js or .android.js extensions
  • Import normally without specifying platform:

// Import automatically selects the right file
import MedicationCard from './MedicationCard';

// Will resolve to:
// - MedicationCard.ios.js on iOS
// - MedicationCard.android.js on Android
// - MedicationCard.js as fallback

Web vs React Native Component Comparison

Web Element React Native Equivalent Key Differences
<div> <View> No direct styling inheritance, no background transparency by default
<span>, <p>, <h1> <Text> All text must be inside a Text component
<img> <Image> Requires explicit dimensions, more loading states
<button> <Pressable>, <TouchableOpacity> Different press states, no built-in styling
<input> <TextInput> Different keyboard types, no built-in validation
<select> <Picker> Platform-specific appearance and behavior
overflow: auto <ScrollView>, <FlatList> Separate components for scrolling
CSS StyleSheet API JavaScript object-based styling with a subset of CSS properties

Summary: Core UI Components

  • React Native provides a set of core components that map to native UI elements
  • View, Text, and Image are the fundamental building blocks
  • ScrollView and FlatList enable scrollable content
  • Pressable allows for creating interactive elements
  • React Native components differ from web components in important ways
  • Platform-specific code can be handled in several ways
