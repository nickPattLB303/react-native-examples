Introduction to React Native Styling

React Native uses a styling approach that is similar to CSS but with some important differences:

  • Styles are written in JavaScript, not CSS
  • Uses a subset of CSS properties with camelCase naming
  • No cascading or inheritance (except for text)
  • Layout primarily uses Flexbox
  • Units are unitless numbers (no px, em, rem, etc.)
Key Insight

React Native styles are not actual CSS. They are JavaScript objects that resemble CSS, but with limitations and differences in behavior.

Emphasize that the mental model from web CSS needs to be adjusted when working with React Native styling.

Style Objects in React Native

The most basic way to style a component is by passing a style object to the component's style prop:


import { View, Text } from 'react-native';

function BasicStyling() {
  return (
    
      
        Amoxicillin 500mg
      
    
  );
}
Performance Note

Inline style objects are recreated on each render, which can impact performance in complex UIs.

Common Style Properties

Layout Properties

  • flex, flexDirection
  • justifyContent, alignItems
  • width, height
  • margin, padding
  • position (relative/absolute)

Visual Properties

  • backgroundColor, color
  • borderRadius, borderWidth
  • opacity, elevation (Android)
  • shadowColor, shadowOffset (iOS)
  • fontWeight, fontSize
Web vs. React Native

Component-Specific Styles

Different components accept different style properties:

View Styles

  • Layout properties (flexbox)
  • Background styles
  • Border styles
  • Shadow properties
  • Transform properties

Text Styles

  • Font properties
  • Text alignment and decoration
  • Line height and letter spacing
  • Text color and shadow
  • Note: Text styles cascade to nested Text
Common Mistake

You cannot apply text styling properties (like fontSize) to a View component.

Combining and Merging Styles

Styles can be combined using array syntax:


function MedicationItem({ highlighted }) {
  const baseStyle = {
    padding: 16,
    borderRadius: 8,
    backgroundColor: '#ffffff'
  };
  
  const highlightStyle = {
    backgroundColor: '#f0f8ff',
    borderWidth: 1,
    borderColor: '#007aff'
  };
  
  return (
    
      Amoxicillin 500mg
    
  );
}
Style Precedence

Later styles in the array override earlier ones with the same properties.

Platform-Specific Styles

Apply different styles based on platform using the Platform API:


import { View, StyleSheet, Platform } from 'react-native';

function MedicationCard() {
  return (
    
      {/* Card contents */}
    
  );
}

const styles = StyleSheet.create({
  card: {
    padding: 16,
    borderRadius: 8,
    backgroundColor: 'white',
    margin: 8,
    // Platform-specific shadow styles
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.2,
        shadowRadius: 4,
      },
      android: {
        elevation: 4,
      },
    }),
  },
});

Style Limitations in React Native

Not Supported in React Native

  • CSS selectors (:hover, :active, etc.)
  • CSS animations and transitions
  • Grid layout
  • Most CSS units (rem, em, vh, etc.)
  • Automatic styling inheritance
  • Media queries (handled differently)

React Native Alternatives

  • State-based styling
  • Animated API
  • Flexbox for layouts
  • Dimensions API for responsive design
  • Explicit style propagation
  • Programmatic adaptation

Using Styled Components in React Native

Styled-components provides a more CSS-like approach to styling React Native:


import styled from 'styled-components/native';

// Create styled components
const CardContainer = styled.View`
  padding: 16px;
  border-radius: 8px;
  background-color: white;
  margin: 8px;
  ${Platform.OS === 'ios' ? `
    shadow-color: #000;
    shadow-offset: 0px 2px;
    shadow-opacity: 0.2;
    shadow-radius: 4px;
  ` : `
    elevation: 4;
  `}
`;

const MedicationName = styled.Text`
  font-size: 18px;
  font-weight: bold;
  color: #333;
  margin-bottom: 8px;
`;

// Use the components
function MedicationCard({ name }) {
  return (
    
      {name}
      {/* Other content */}
    
  );
}

Under the Hood: How React Native Styling Works

The JavaScript Thread

  • Style objects are defined in JS
  • StyleSheet.create optimizes by creating style IDs
  • Style resolution happens in JS
  • Resolved styles sent across the bridge

The Native UI Thread

  • Receives style information from JS
  • Converts to platform-specific styles
  • iOS: YogaKit + UIKit styling
  • Android: Yoga + View styling
Performance Insight

The JavaScript-to-Native bridge can be a performance bottleneck. Using StyleSheet.create() creates IDs for styles rather than sending full style objects across the bridge repeatedly.

Styling Best Practices

Performance

  • Use StyleSheet.create instead of inline styles
  • Avoid complex style computations in render
  • Memoize dynamic styles when possible
  • Keep style objects outside component functions

Organization

  • Group related styles together
  • Use meaningful style names
  • Separate component logic from styling
  • Consider a theming approach for consistency

Section Summary

  • React Native uses JavaScript objects for styling, not CSS
  • Styles can be inline or created with StyleSheet.create()
  • Different components accept different style properties
  • Styles can be combined, merged, and conditionally applied
  • Platform.select() allows for platform-specific styling
  • Styled-components provides an alternative CSS-like styling approach
