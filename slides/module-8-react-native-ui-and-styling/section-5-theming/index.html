What are Themes and Design Systems?

Themes

  • A collection of styles and visual properties
  • Define the look and feel of an application
  • Can be switched dynamically (e.g., dark/light mode)
  • Provide consistent visual experience
  • Centralize style definitions

Design Systems

  • A comprehensive set of UI components
  • Includes styling guidelines and principles
  • Defines patterns for composition
  • Ensures consistency across the application
  • Improves developer velocity and UX quality
Key Insight

Design systems help teams build consistent user interfaces efficiently while themes allow for visual customization and adaptation.

Design Tokens: Building Blocks of Design Systems

Design tokens are named values that store visual design attributes:

Common Design Tokens

  • Colors: primary, secondary, accent
  • Typography: font families, sizes, weights
  • Spacing: margins, paddings, gaps
  • Borders: widths, radii, colors
  • Shadows: elevations, depths

Creating a Theme Provider with React Context

Use React Context to make theme available throughout your application:


// theme/ThemeContext.js
import React, { createContext, useContext, useState } from 'react';
import { lightTheme, darkTheme } from './themes';

// Create context with default value
const ThemeContext = createContext({
  theme: lightTheme,
  isDark: false,
  toggleTheme: () => {},
});

// Create provider component
export function ThemeProvider({ children }) {
  const [isDark, setIsDark] = useState(false);
  const theme = isDark ? darkTheme : lightTheme;
  
  // Toggle between light and dark themes
  const toggleTheme = () => {
    setIsDark(!isDark);
  };
  
  return (
    
      {children}
    
  );
}

// Custom hook for accessing theme
export function useTheme() {
  const context = useContext(ThemeContext);
  if (context === undefined) {
    throw new Error('useTheme must be used within a ThemeProvider');
  }
  return context;
}

Creating Theme-Aware Components

Consume the theme context in your components:


// components/Button.js
import React from 'react';
import { TouchableOpacity, Text, StyleSheet } from 'react-native';
import { useTheme } from '../theme/ThemeContext';

export function Button({ title, variant = 'primary', onPress, disabled }) {
  const { theme } = useTheme();
  
  // Determine button style based on variant and theme
  const getButtonStyle = () => {
    switch (variant) {
      case 'primary':
        return {
          backgroundColor: theme.colors.primary,
          borderColor: theme.colors.primary,
        };
      case 'secondary':
        return {
          backgroundColor: 'transparent',
          borderColor: theme.colors.primary,
          borderWidth: 1,
        };
      case 'outline':
        return {
          backgroundColor: 'transparent',
          borderColor: theme.colors.border,
          borderWidth: 1,
        };
      default:
        return {};
    }
  };
  
  // Determine text style based on variant and theme
  const getTextStyle = () => {
    switch (variant) {
      case 'primary':
        return {
          color: '#FFFFFF',
        };
      case 'secondary':
      case 'outline':
        return {
          color: theme.colors.primary,
        };
      default:
        return {};
    }
  };
  
  return (
    
      
        {title}
      
    
  );
}

const styles = StyleSheet.create({
  button: {
    borderRadius: 8,
    alignItems: 'center',
    justifyContent: 'center',
  },
  text: {
    fontWeight: '600',
  },
  disabled: {
    opacity: 0.6,
  },
  disabledText: {
    opacity: 0.8,
  },
});

Supporting System Dark Mode

React to system theme changes with the useColorScheme hook:


// theme/ThemeContext.js (updated)
import { useColorScheme } from 'react-native';

export function ThemeProvider({ children }) {
  // Get system color scheme
  const systemColorScheme = useColorScheme();
  
  // Initialize with system preference
  const [themeMode, setThemeMode] = useState(systemColorScheme || 'light');
  
  // Update when system preference changes
  useEffect(() => {
    if (systemColorScheme) {
      setThemeMode(systemColorScheme);
    }
  }, [systemColorScheme]);
  
  const isDark = themeMode === 'dark';
  const theme = isDark ? darkTheme : lightTheme;
  
  // Methods to control theme
  const setDarkTheme = () => setThemeMode('dark');
  const setLightTheme = () => setThemeMode('light');
  const setSystemTheme = () => setThemeMode(systemColorScheme);
  
  const value = {
    theme,
    isDark,
    themeMode,
    setDarkTheme,
    setLightTheme,
    setSystemTheme,
    toggleTheme: () => setThemeMode(isDark ? 'light' : 'dark'),
  };
  
  return (
    
      {children}
    
  );
}
Note

The useColorScheme hook is only available in React Native 0.62 and above.

export function ThemeProvider({ children, initialTheme = 'light' }) {
  const [currentTheme, setCurrentTheme] = useState(initialTheme);

Third-Party Theming Libraries

Library Selection Criteria

  • Component library needs
  • Customization requirements
  • Design system compatibility
  • TypeScript support
  • Bundle size considerations
  • Active maintenance
Decision Point

Consider if you need a full UI library with theming or just a theming system for custom components.

Building a Complete Design System

Structure of a comprehensive React Native design system:

Core Components

  • Layout: Container, Box, Grid
  • Typography: Text, Heading
  • Inputs: Button, Input, Select
  • Feedback: Alert, Toast, Progress
  • Navigation: Tabs, Drawer
  • Media: Avatar, Icon, Image

Design System Structure


/design-system
  /themes
    - tokens.js        # Design tokens
    - lightTheme.js    # Light theme
    - darkTheme.js     # Dark theme
  /components
    /core              # Primitive components
    /composite         # Composed components
    /layout            # Layout components
  /hooks               # Custom hooks
  /utils               # Utility functions
  index.js             # Public API
Implementation Note

Start small with core components and add complexity gradually as your design system matures.

  • Design systems provide a comprehensive set of UI components and guidelines
