Module 10

State Management

React Native Training Course | 2-3 hours

Module Learning Objectives

  • Understand different types of state in React Native applications
  • Implement local state management using React hooks
  • Use React Query for server state management
  • Implement client state management with Zustand
  • Use Context API for shared state
  • Implement data persistence with various storage options
  • Apply best practices for state management

Module Journey

This module takes you on a journey from understanding basic state management concepts to implementing complex state management solutions in your pharmacy application. You'll learn how to manage different types of state using the most appropriate tools and techniques.

Time Management: If running short on time, prioritize Sections 1, 2, and 3 as they provide the foundational understanding and the most commonly used state management solutions. Section 4 can be covered at a higher level, focusing on when to use Context API instead of Zustand.

Target Audience

This module is designed for developers with experience in:

Android Developers

  • Familiar with ViewModel and LiveData
  • Experience with state management in Android
  • Understanding of data persistence

iOS Developers

  • Familiar with SwiftUI state management
  • Experience with Combine or Redux-like patterns
  • Understanding of data persistence

Web Developers

  • Familiar with React state management
  • Experience with Redux, MobX, or similar libraries
  • Understanding of client-server data flow

Course Theme: Pharmacy/Medication App

In this module, we'll implement state management for our pharmacy application:

  • Authentication state for user login and registration
  • Medication data fetching and caching
  • Shopping cart state management
  • User preferences and settings
  • Offline data persistence
State Management Requirements

Our pharmacy app requires a comprehensive state management system that handles server data, client state, and persistent storage. The state management should be efficient, maintainable, and provide a good user experience even in offline scenarios.

Module Challenge

At the end of this module, you'll complete a challenge to implement a comprehensive state management system for a medication tracking application:

  1. Implement server state management using React Query
  2. Implement client state management using Zustand
  3. Use Context API for shared state
  4. Implement data persistence for offline capabilities
  5. Create a scalable and maintainable state management architecture
Challenge Preparation

Take notes throughout the module on state management patterns and implementation details that will help you complete the challenge. Pay special attention to the integration between different state management solutions and offline capabilities.
