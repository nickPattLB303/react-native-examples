List Components: FlatList

Key Features

  • Performance optimized for large lists
  • Only renders items currently visible ("windowing")
  • Built-in support for pull-to-refresh
  • Handles item separators elegantly
  • Supports headers and footers
  • Easy pagination with onEndReached

Key Properties

  • data: Array of items to display
  • renderItem: Function to render each item
  • keyExtractor: Function to extract unique keys
  • ItemSeparatorComponent: Component between items
  • ListHeaderComponent: Header component
  • ListFooterComponent: Footer component
  • onEndReached: Called when end of list is reached
import React from 'react';

      {item.name}

    />
  );
};
Performance Tip

Always provide a keyExtractor that returns a stable, unique string for each item to optimize rendering performance.

Pharmaceutical Applications

import React, { useState } from 'react';
import { View, TextInput, Text, StyleSheet } from 'react-native';

const ValidatedInput = ({ 
  label, 
  placeholder, 
  value, 
  onChangeText, 
  validator, 
  errorMessage,
  keyboardType = 'default',
  secureTextEntry = false,
}) => {
  const [touched, setTouched] = useState(false);
  const isValid = !validator || validator(value);
  const showError = touched && !isValid;

  return (
    
      {label}
       setTouched(true)}
        keyboardType={keyboardType}
        secureTextEntry={secureTextEntry}
      />
      {showError && (
        {errorMessage}
      )}
    
  );
};
Example Usage
// For dosage validation
const validateDosage = (value) => {

  return /^\d+(\.\d+)?\s*(mg|ml|g|mcg)$/i.test(value);
};

Pharmaceutical Applications

  • Medication reminder toggles
  • Privacy settings
  • Feature toggles for medication tracking
import React from 'react';
import { TouchableOpacity, View, Text, 
  Animated, StyleSheet } from 'react-native';

const CustomSwitch = ({ 
  value, 
  onValueChange, 
  label,
  disabled = false,
  activeColor = '#4CAF50',
  inactiveColor = '#f5f5f5'
}) => {
  // Animation value for the position of the thumb
  const thumbPosition = React.useRef(
    new Animated.Value(value ? 28 : 2)
  ).current;
  
  // Animation function
  const toggleSwitch = () => {
    if (disabled) return;
    
    Animated.timing(thumbPosition, {
      toValue: value ? 2 : 28,
      duration: 250,
      useNativeDriver: true,
    }).start();
    
    onValueChange(!value);
  };
  
  return (
    
      {label}
      
      
        
      
    
  );
};
Animation Tip

Use the useNativeDriver: true option when possible to offload animations to the native thread for better performance.

Creating a Component Library: Button Component

Button Variants

  • Contained: Solid background color
  • Outlined: Border with transparent background
  • Text: Text-only button

Button States

  • Normal: Default appearance
  • Pressed: Visual feedback when touched
  • Disabled: Indicates non-interactive state
  • Loading: Shows activity indicator

Button Props

  • title: Button label text
  • onPress: Press handler function
  • variant: Visual style variant
  • color: Theme color to use
  • size: Button size (small, medium, large)
  • disabled: Whether button is interactive
  • loading: Whether to show loading state
  • fullWidth: Whether to stretch to container width
// components/Button.js
import React from 'react';
import { TouchableOpacity, Text, StyleSheet, 
  ActivityIndicator } from 'react-native';
import { colors, typography, spacing, 
  borderRadius } from '../theme';

const Button = ({
  title,
  onPress,
  variant = 'contained', // 'contained', 'outlined', 'text'
  color = 'primary', // 'primary', 'secondary', 'error'
  size = 'medium', // 'small', 'medium', 'large'
  disabled = false,
  loading = false,
  fullWidth = false,
  style,
  textStyle,
}) => {
  const getButtonStyle = () => {
    let buttonStyle = [styles.button, styles[size]];
    
    if (variant === 'contained') {
      buttonStyle.push({
        backgroundColor: colors[color],
      });
    } else if (variant === 'outlined') {
      buttonStyle.push({
        backgroundColor: 'transparent',
        borderWidth: 1,
        borderColor: colors[color],
      });
    }
    
    if (fullWidth) {
      buttonStyle.push(styles.fullWidth);
    }
    
    if (disabled) {
      buttonStyle.push(styles.disabled);
    }
    
    if (style) {
      buttonStyle.push(style);
    }
    
    return buttonStyle;
  };
  
  // Return TouchableOpacity with appropriate styles
  // and loading indicator or text
};

Best Practices for UI Components

1. Create Consistent Theme

Define color palettes, typography, spacing, and border radius constants for consistent visual language.

2. Build Reusable Components

Create a library of reusable components with clear props API and consistent behavior.

3. Separate Logic from Presentation

Use container/presenter pattern or custom hooks to separate business logic from UI components.

4. Accessibility

Make all components accessible with appropriate props for screen readers and support for larger text sizes.

5. Performance

Optimize render performance with memoization, virtualized lists, and proper key management.

6. Platform Adaptation

Provide platform-specific variants when appropriate to match native platform conventions.

7. Documentation

Document component usage, props, and examples to make them easier for your team to use.
