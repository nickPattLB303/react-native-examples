Styling in React Native

React Native uses JavaScript objects for styling instead of CSS:

Key Differences from CSS

  • Use StyleSheet.create() to define styles
  • Camel-cased property names (e.g., backgroundColor)
  • No CSS selectors or cascade
  • No inheritance except in Text components
  • No media queries (use Dimensions API instead)
  • Limited set of supported properties
import { StyleSheet, View, Text } from 'react-native';

// Basic component with styling
const MedicationLabel = () => (
  <View style={styles.container}>
    <Text style={styles.drugName}>Metformin</Text>
    <Text style={styles.dosage}>500mg</Text>
    <Text style={styles.instructions}>
      Take 1 tablet twice daily with meals.
    </Text>
  </View>
);

// StyleSheet definition
const styles = StyleSheet.create({
  container: {
    padding: 16,
    borderRadius: 8,
    backgroundColor: 'white',
    shadowColor: '#000',
    shadowOpacity: 0.1,
    shadowRadius: 4,
    shadowOffset: { width: 0, height: 2 },
    elevation: 2,
  },
  drugName: {
    fontSize: 18,
    fontWeight: 'bold',
    color: '#333',
  },
  dosage: {
    fontSize: 16,
    color: '#666',
    marginTop: 4,
  },
  instructions: {
    fontSize: 14,
    marginTop: 12,
    lineHeight: 20,
  },
});
StyleSheet Benefits

Using StyleSheet.create() provides validation, optimization, and better memory management compared to inline object literals.

Style Property Types

React Native supports various style properties, grouped by category:

Layout Properties

  • Width/Height: width, height
  • Padding: padding, paddingTop...
  • Margin: margin, marginLeft...
  • Position: position, top...
  • Flexbox: flex, flexDirection...

Visual Properties

  • Colors: backgroundColor, color
  • Borders: borderWidth, borderColor
  • Border Radius: borderRadius
  • Opacity: opacity
  • Shadows: shadowColor, elevation...

Typography Properties

  • Font Styling: fontSize, fontWeight
  • Text Alignment: textAlign
  • Line Height: lineHeight
  • Letter Spacing: letterSpacing
  • Font Families: fontFamily

Image Properties

  • Resize Mode: resizeMode
  • Tint Color: tintColor
  • Object Fit: objectFit
  • Background Position: backgroundPosition
Important Difference

Dimensions in React Native are unitless and represent density-independent pixels. Don't add units like 'px' or '%' to numeric values.

Inline vs StyleSheet

There are multiple ways to apply styles to React Native components:

Inline Styles

<View 
  style={{ 
    padding: 16,
    backgroundColor: 'white',
    borderRadius: 8, 
  }}
>
  <Text 
    style={{ 
      fontSize: 16, 
      fontWeight: 'bold', 
      color: '#333' 
    }}
  >
    Ibuprofen
  </Text>
</View>

Pros: Simple for small components, dynamic styles

Cons: Performance issues, no validation, cluttered code

StyleSheet API

const styles = StyleSheet.create({
  container: {
    padding: 16,
    backgroundColor: 'white',
    borderRadius: 8,
  },
  title: {
    fontSize: 16,
    fontWeight: 'bold',
    color: '#333',
  },
});

<View style={styles.container}>
  <Text style={styles.title}>Ibuprofen</Text>
</View>

Pros: Better performance, validation, separation of concerns

Cons: Less flexible for dynamic styles

Best Practice

Use StyleSheet.create() for static styles and inline styles for dynamic values or when merging styles.

Style Inheritance

React Native's style inheritance works differently than CSS:

No Cascading Styles

// DOESN'T WORK
<View style={{ backgroundColor: 'blue' }}>
  {/* Color does NOT inherit to child View */}
  <View>
    <Text>This text is not blue</Text>
  </View>
</View>

Most styles do not cascade to children.

View Styles

  • Layout properties apply only to the component they're set on
  • Children components need their own styles
  • Parent containers only control layout, not visual properties

Text Component Exception

// DOES WORK
<Text style={{ color: 'blue', fontSize: 16 }}>
  Blue text
  <Text style={{ fontWeight: 'bold' }}>
    Blue AND bold text
  </Text>
</Text>

Text components do inherit styles to nested Text children.

Common Inheritance Pattern

// Pharmacy app example
<Text style={styles.instructions}>
  Take 
  <Text style={styles.emphasis}> two tablets </Text>
  daily with water.
</Text>

const styles = StyleSheet.create({
  instructions: {
    fontSize: 14,
    color: '#333',
  },
  emphasis: {
    fontWeight: 'bold',
    color: '#e74c3c',
  },
});
Common Mistake

Placing Text directly inside a View without a style won't inherit any styles from parent Views.

Style Composition

React Native allows you to combine multiple styles in different ways:

Using Arrays

// Basic array composition
<View style={[styles.container, styles.card]}>
  <Text style={styles.text}>Aspirin</Text>
</View>

// With conditional styles
<View 
  style={[
    styles.pill,
    isActive && styles.activePill,
    isPrescription && styles.prescriptionPill
  ]}
>
  <Text>Medication</Text>
</View>

Later styles in the array override earlier ones.

Spreading Objects

// Combining style objects
<View 
  style={{
    ...styles.base,
    ...styles.card,
    backgroundColor: isDanger ? 'red' : 'white',
  }}
>
  <Text>Warning</Text>
</View>

// Creating dynamic styles
const dynamicStyles = {
  height: isExpanded ? 200 : 50,
  opacity: isVisible ? 1 : 0.5,
};

<View style={[styles.container, dynamicStyles]}>
  <Text>Dynamic content</Text>
</View>
Style Precedence

When styles conflict, the last style in an array or spread object takes precedence.

Style Organization Patterns

As your app grows, organizing styles becomes important:

Component Styles

// Component-specific styles
const MedicationCard = ({ medication }) => (
  <View style={styles.card}>
    <Text style={styles.name}>{medication.name}</Text>
    <Text style={styles.dosage}>{medication.dosage}</Text>
  </View>
);

// Styles defined in the same file
const styles = StyleSheet.create({
  card: { /* styles */ },
  name: { /* styles */ },
  dosage: { /* styles */ },
});

Style Modules

// styles/colors.js
export default {

Theme System

For larger apps, consider creating a theme system with consistent colors, spacing, and typography that can be used across components.

Conditional Styling

Apply styles dynamically based on props, state, or other conditions:

Using Logical AND

// Using logical AND
<View 
  style={[
    styles.medicationStatus,
    isActive && styles.activeMedication,
  ]}
>
  <Text>Status: {isActive ? 'Active' : 'Inactive'}</Text>
</View>

// Only applies activeMedication when isActive is true

Using Ternary Operator

// Using ternary for styles
<Text 
  style={[
    styles.dosage,
    isDosageHigh 
      ? styles.highDosage 
      : styles.normalDosage
  ]}
>
  {medication.dosage}
</Text>

Style Functions

// Generating styles from props
const MedicationBadge = ({ status }) => {
  const getBadgeStyle = () => {
    switch (status) {
      case 'active':
        return styles.activeBadge;
      case 'expired':
        return styles.expiredBadge;
      case 'lowSupply':
        return styles.lowSupplyBadge;
      default:
        return styles.defaultBadge;
    }
  };
  
  return (
    <View style={[styles.badge, getBadgeStyle()]}>
      <Text style={styles.badgeText}>{status}</Text>
    </View>
  );
};

Pressable's Style Function

// Using Pressable's style function
<Pressable
  style={({ pressed }) => [
    styles.button,
    pressed && styles.buttonPressed
  ]}
>
  <Text style={styles.buttonText}>Take Medication</Text>
</Pressable>
Performance Tip

For complex conditional styles, compute them once instead of recreating style objects on every render.

Platform-Specific Styles

Apply different styles for iOS and Android:

Platform Module

import { Platform, StyleSheet } from 'react-native';

const styles = StyleSheet.create({
  medicationCard: {
    padding: 16,
    borderRadius: 8,
    backgroundColor: 'white',
    ...Platform.select({
      ios: {
        shadowColor: '#000',
        shadowOffset: { width: 0, height: 2 },
        shadowOpacity: 0.1,
        shadowRadius: 4,
      },
      android: {
        elevation: 3,
      },
    }),
  },
  
  button: {
    // Platform-specific values
    paddingVertical: Platform.OS === 'ios' ? 12 : 8,
    marginTop: Platform.OS === 'ios' ? 16 : 12,
  },
});

Platform-Specific Extensions

// MedicationButton.ios.js
import React from 'react';
import { TouchableOpacity, Text, StyleSheet } from 'react-native';

export default function MedicationButton({ title, onPress }) {
  return (
    <TouchableOpacity style={styles.iosButton} onPress={onPress}>
      <Text style={styles.iosButtonText}>{title}</Text>
    </TouchableOpacity>
  );
}

// MedicationButton.android.js
import React from 'react';
import { TouchableNativeFeedback, View, Text, StyleSheet } from 'react-native';

export default function MedicationButton({ title, onPress }) {
  return (
    <TouchableNativeFeedback onPress={onPress}>
      <View style={styles.androidButton}>
        <Text style={styles.androidButtonText}>{title}</Text>
      </View>
    </TouchableNativeFeedback>
  );
}

Import as: import MedicationButton from './MedicationButton';
React Native will automatically select the right file.

When to Use

Use platform-specific styling when you want to match native UI conventions or need to handle platform differences in shadows, buttons, etc.

Common Style Issues

Text Overflow

// Handling text overflow
<Text 
  numberOfLines={2}
  ellipsizeMode="tail"
  style={styles.medicationName}
>
  This is a very long medication name that might not fit
</Text>

// Long text will show as: "This is a very long medication..."

Absolute Positioning

// Absolute positioning
const styles = StyleSheet.create({
  container: {
    position: 'relative', // Parent needs relative
    height: 100,
  },
  badge: {
    position: 'absolute',
    top: -10,
    right: -10,
    backgroundColor: 'red',
    width: 20,
    height: 20,
    borderRadius: 10,
  },
});

Shadow Issues

// Cross-platform shadows
const styles = StyleSheet.create({
  card: {
    backgroundColor: 'white',
    borderRadius: 8,
    padding: 16,
    // iOS shadows
    shadowColor: '#000',
    shadowOffset: { width: 0, height: 2 },
    shadowOpacity: 0.1,
    shadowRadius: 4,
    // Android shadow
    elevation: 3,
  },
});

Centering Content

// Centering content
const styles = StyleSheet.create({
  // Horizontal and vertical centering
  centered: {
    flex: 1,
    justifyContent: 'center',
    alignItems: 'center',
  },
  // Horizontal centering only
  horizontallyCentered: {
    alignItems: 'center',
  },
});
Common Gotcha

Remember that shadow properties only work on iOS and elevation only works on Android. You need to use both for cross-platform shadows.

Style Performance Optimization

Performance tips for styling in React Native:

  • Use StyleSheet.create(): It's optimized internally to make IDs for faster lookup
  • Avoid Dynamic Styles: Don't recreate style objects on every render
  • Memoize Dynamic Styles: Use useMemo for styles that depend on props/state
  • Flatten StyleSheet: For critical performance, use StyleSheet.flatten()
  • Avoid Deep Nesting: Flatten your component hierarchy when possible
  • Reuse Style Objects: Define shared styles in one place
// Memoizing dynamic styles example
import React, { useMemo } from 'react';
import { View, Text, StyleSheet } from 'react-native';

const DosageIndicator = ({ dosage, maxDosage }) => {
  // Calculate percentage
  const percentage = Math.min(100, (dosage / maxDosage) * 100);
  
  // Memoize dynamic styles
  const dynamicStyles = useMemo(() => ({
    indicator: {
      width: `${percentage}%`,
      backgroundColor: percentage > 80 ? '#e74c3c' : '#3498db',
    },
  }), [percentage]);
  
  return (
    <View style={styles.container}>
      <View style={[styles.track]}>
        <View style={[styles.baseline, dynamicStyles.indicator]} />
      </View>
      <Text style={styles.text}>{dosage} / {maxDosage} mg</Text>
    </View>
  );
};
Performance Tip

For lists, make sure your style objects are created outside the render function to prevent recreation on each list item render.

Styling Best Practices

Organization

  • Separate concerns: Keep styles separate from component logic
  • Consistent naming: Use clear, consistent style naming conventions
  • Create a theme: Centralize colors, spacing, and typography
  • Group related styles: Organize styles by component or feature

Maintainability

  • Avoid magic numbers: Use constants for important values
  • Comment complex styles: Explain non-obvious style choices
  • Use descriptive names: headerContainer vs topView
  • Reuse styles: Create common style objects for repeated patterns

Implementation

  • Use StyleSheet.create(): Don't use inline styles for static UI
  • Compose with arrays: Use style arrays for composition
  • Test on multiple devices: Ensure styles work across screen sizes
  • Handle platform differences: Use Platform.select or platform-specific files

Performance

  • Flatten styles: Don't nest components unnecessarily
  • Memoize dynamic styles: Use useMemo for styles that change
  • Avoid recreating styles: Define styles outside render methods
  • Use opacity/transforms: These properties are hardware-accelerated
Pharmaceutical App Tips

For medication apps, maintain strict consistency in colors for different medication types or statuses to help users quickly identify critical information. Focus on accessibility with proper contrast ratios and readable text sizes.

Summary

  • React Native uses JavaScript objects for styling instead of CSS stylesheets
  • StyleSheet API offers optimizations and validation over inline styles
  • Style properties in React Native are similar to CSS but use camelCase syntax
  • Style inheritance only works within nested Text components
  • Styles can be composed using arrays and spread operators
  • Conditional styling can be applied using logical operators or style functions
  • Platform-specific styling is essential for matching native UI expectations
  • Good style organization improves maintainability and development speed
