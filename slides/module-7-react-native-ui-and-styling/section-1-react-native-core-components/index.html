React Native Core Components

React Native provides a set of built-in components that map directly to native UI elements:

Basic Components

  • View: Basic container (like div)
  • Text: Display text (like span/p)
  • Image: Display images
  • TextInput: Text input fields
  • ScrollView: Scrollable container

Interactive Components

  • Button: Simple button
  • TouchableOpacity: Touchable with opacity feedback
  • TouchableHighlight: Touchable with highlight feedback
  • Pressable: Flexible touchable area (newer API)
  • Switch: Toggle switch
Component Philosophy

Unlike the web, React Native has a limited set of core components. This is intentional—these components map directly to native UI elements on iOS and Android.

The View Component

View is the fundamental building block in React Native, similar to div in web:

Key Properties

  • style: Styling (layout, color, etc.)
  • onLayout: Event when layout changes
  • pointerEvents: Control touch handling
  • testID: For UI testing
  • accessibilityLabel: For accessibility
import { View, StyleSheet } from 'react-native';

const MedicationContainer = () => {
  return (
    <View style={styles.container}>
      <View style={styles.card}>
        {/* Medication info goes here */}
      </View>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
    padding: 16,
    backgroundColor: '#f9f9f9',
  },
  card: {
    backgroundColor: 'white',
    borderRadius: 8,
    padding: 16,
    shadowColor: '#000',
    shadowOpacity: 0.1,
    shadowRadius: 4,
    elevation: 2,
  },
});
Common Mistake

Unlike web divs, Views do not have any intrinsic styling. They don't support text content directly—text must be inside a Text component.

The Text Component

Text is used for displaying text content in React Native:

Key Properties

  • style: Typography styling
  • numberOfLines: Truncate text
  • ellipsizeMode: How to truncate
  • selectable: Allow text selection
  • onPress: Handle text press
import { Text, StyleSheet, View } from 'react-native';

const MedicationHeader = ({ name, dosage }) => {
  return (
    <View style={styles.header}>
      <Text style={styles.title} numberOfLines={1}>
        {name}
      </Text>
      <Text style={styles.dosage}>
        {dosage}
      </Text>
      <Text style={styles.instruction} numberOfLines={2}>
        Take with food. Avoid alcohol while using this medication.
      </Text>
    </View>
  );
};

const styles = StyleSheet.create({
  header: {
    marginBottom: 16,
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    color: '#333',
  },
  dosage: {
    fontSize: 18,
    color: '#666',
    marginBottom: 8,
  },
  instruction: {
    fontSize: 14,
    color: '#888',
    fontStyle: 'italic',
  },
});
Text Nesting

Unlike web, Text components can be nested to inherit and override styles, which is useful for inline styling.

The Image Component

Image displays different types of images in React Native:

Key Properties

  • source: URI or local image source
  • style: Style the image (width, height)
  • resizeMode: How to resize (cover, contain, etc.)
  • onLoad/onError: Image loading events
  • blurRadius: Add blur effect

Image Sources

  • Local assets: require('./image.png')
  • Remote URLs: {{ uri: 'https://...' }}
  • Base64: {{ uri: 'data:image/png;base64,..'}}
import { Image, StyleSheet, View } from 'react-native';

const MedicationImage = ({ imageUri }) => {
  return (
    <View style={styles.imageContainer}>
      <Image
        source={{ uri: imageUri }}
        style={styles.image}
        resizeMode="cover"
      />
    </View>
  );
};

// Using local image
const MedicationIcon = () => (
  <Image
    source={require('../assets/pill-icon.png')}
    style={styles.icon}
  />
);

const styles = StyleSheet.create({
  imageContainer: {
    alignItems: 'center',
    marginBottom: 16,
  },
  image: {
    width: 150,
    height: 150,
    borderRadius: 75,
  },
  icon: {
    width: 24,
    height: 24,
  },
});
Important

Unlike the web, Image components require explicit width and height in styles, or they won't be visible.

The TextInput Component

TextInput allows users to input text in React Native:

Key Properties

  • value/onChangeText: Control input value
  • placeholder: Placeholder text
  • secureTextEntry: For passwords
  • keyboardType: Type of keyboard
  • multiline: Allow multiple lines
  • autoCapitalize: Auto-capitalization
  • autoCorrect: Auto-correction
import React, { useState } from 'react';
import { TextInput, StyleSheet, View, Text } from 'react-native';

const MedicationSearch = () => {
  const [searchText, setSearchText] = useState('');
  
  return (
    <View style={styles.container}>
      <Text style={styles.label}>Search Medications:</Text>
      <TextInput
        style={styles.input}
        value={searchText}
        onChangeText={setSearchText}
        placeholder="Enter medication name"
        placeholderTextColor="#999"
        autoCapitalize="none"
        autoCorrect={false}
        clearButtonMode="while-editing"
      />
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    marginBottom: 16,
  },
  label: {
    fontSize: 16,
    fontWeight: '500',
    marginBottom: 8,
  },
  input: {
    height: 48,
    borderWidth: 1,
    borderColor: '#ddd',
    borderRadius: 8,
    paddingHorizontal: 16,
    fontSize: 16,
  },
});
Platform Differences

TextInput behavior and appearance can vary between iOS and Android. Test on both platforms and use platform-specific props when needed.

Touchable Components

React Native provides several components for creating touchable elements:

Touchable Types

  • Button: Simple button with platform styling
  • TouchableOpacity: Fades when pressed
  • TouchableHighlight: Highlights when pressed
  • TouchableWithoutFeedback: No visual feedback
  • Pressable: Modern API with more control

Common Props

  • onPress: Function when pressed
  • onLongPress: Function on long press
  • disabled: Disable touch events
import React from 'react';
import {
  Button,
  TouchableOpacity,
  TouchableHighlight,
  Pressable,
  Text,
  StyleSheet,
  View
} from 'react-native';

const MedicationActions = ({ onTake, onRefill }) => {
  return (
    <View style={styles.container}>
      {/* Simple Button */}
      <Button
        title="Refill Medication"
        onPress={onRefill}
        color="#3498db"
      />
      
      {/* TouchableOpacity */}
      <TouchableOpacity
        style={styles.takeButton}
        onPress={onTake}
        activeOpacity={0.7}
      >
        <Text style={styles.buttonText}>Take Medication</Text>
      </TouchableOpacity>
      
      {/* Modern Pressable */}
      <Pressable
        style={({ pressed }) => [
          styles.infoButton,
          pressed && styles.buttonPressed
        ]}
        onPress={() => console.log('Info pressed')}
      >
        <Text style={styles.infoText}>View Information</Text>
      </Pressable>
    </View>
  );
};

const styles = StyleSheet.create({
  container: {
    gap: 12,
  },
  takeButton: {
    backgroundColor: '#2ecc71',
    padding: 16,
    borderRadius: 8,
    alignItems: 'center',
  },
  buttonText: {
    color: 'white',
    fontWeight: 'bold',
    fontSize: 16,
  },
  infoButton: {
    borderWidth: 1,
    borderColor: '#3498db',
    padding: 12,
    borderRadius: 8,
    alignItems: 'center',
  },
  buttonPressed: {
    backgroundColor: '#e6f7ff',
  },
  infoText: {
    color: '#3498db',
    fontSize: 14,
  },
});
Best Practice

Pressable is the newest and most flexible touchable component, offering the most control over interaction states.

ScrollView Component

ScrollView provides scrolling capabilities for content that exceeds screen size:

Key Properties

  • horizontal: Horizontal scrolling
  • showsVerticalScrollIndicator: Show scroll bars
  • contentContainerStyle: Style the content
  • refreshControl: Add pull-to-refresh
  • pagingEnabled: Page-based scrolling
  • keyboardShouldPersistTaps: Keyboard behavior
import React from 'react';
import {
  ScrollView,
  StyleSheet,
  View,
  Text,
  RefreshControl
} from 'react-native';

const MedicationDetails = ({ medication }) => {
  const [refreshing, setRefreshing] = React.useState(false);

  const onRefresh = React.useCallback(() => {
    setRefreshing(true);
    // Fetch updated medication data
    setTimeout(() => setRefreshing(false), 2000);
  }, []);

  return (
    <ScrollView
      style={styles.container}
      contentContainerStyle={styles.content}
      showsVerticalScrollIndicator={false}
      refreshControl={
        <RefreshControl
          refreshing={refreshing}
          onRefresh={onRefresh}
          colors={['#3498db']}
        />
      }
    >
      <Text style={styles.name}>{medication.name}</Text>
      <Text style={styles.dosage}>{medication.dosage}</Text>
      
      <View style={styles.section}>
        <Text style={styles.sectionTitle}>Description</Text>
        <Text>{medication.description}</Text>
      </View>
      
      <View style={styles.section}>
        <Text style={styles.sectionTitle}>Side Effects</Text>
        <Text>{medication.sideEffects}</Text>
      </View>
      
      <View style={styles.section}>
        <Text style={styles.sectionTitle}>Precautions</Text>
        <Text>{medication.precautions}</Text>
      </View>
    </ScrollView>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
  content: {
    padding: 16,
  },
  name: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 4,
  },
  dosage: {
    fontSize: 18,
    color: '#666',
    marginBottom: 16,
  },
  section: {
    marginBottom: 16,
  },
  sectionTitle: {
    fontSize: 18,
    fontWeight: '600',
    marginBottom: 8,
    color: '#333',
  },
});
Performance Note

ScrollView renders all its content at once, which can lead to performance issues with large lists. For long lists, use FlatList instead.

FlatList Component

FlatList efficiently renders large lists by only rendering items visible on screen:

Key Properties

  • data: Array of data to render
  • renderItem: Render each item
  • keyExtractor: Generate unique keys
  • ItemSeparatorComponent: Render separators
  • ListHeaderComponent/Footer: Header/footer
  • onRefresh/refreshing: Pull to refresh
  • onEndReached: Called when end reached
import React from 'react';
import {
  FlatList,
  StyleSheet,
  View,
  Text,
  TouchableOpacity
} from 'react-native';

const MedicationList = ({ medications, onSelect }) => {
  const renderItem = ({ item }) => (
    <TouchableOpacity
      style={styles.item}
      onPress={() => onSelect(item)}
    >
      <View>
        <Text style={styles.name}>{item.name}</Text>
        <Text style={styles.dosage}>{item.dosage}</Text>
        <Text style={styles.schedule}>{item.schedule}</Text>
      </View>
    </TouchableOpacity>
  );
  
  const renderSeparator = () => (
    <View style={styles.separator} />
  );
  
  const renderHeader = () => (
    <View style={styles.header}>
      <Text style={styles.headerText}>Your Medications</Text>
    </View>
  );

  return (
    <FlatList
      data={medications}
      renderItem={renderItem}
      keyExtractor={(item) => item.id}
      ItemSeparatorComponent={renderSeparator}
      ListHeaderComponent={renderHeader}
      contentContainerStyle={styles.list}
    />
  );
};

const styles = StyleSheet.create({
  list: {
    padding: 16,
  },
  header: {
    marginBottom: 16,
  },
  headerText: {
    fontSize: 22,
    fontWeight: 'bold',
  },
  item: {
    paddingVertical: 12,
  },
  name: {
    fontSize: 18,
    fontWeight: '600',
  },
  dosage: {
    fontSize: 16,
    color: '#666',
  },
  schedule: {
    fontSize: 14,
    color: '#888',
    marginTop: 4,
  },
  separator: {
    height: 1,
    backgroundColor: '#eee',
  },
});
When to Use FlatList

Use FlatList when rendering lists with many items (10+) or an unknown number of items (like data from an API).

Other Important Components

ActivityIndicator

Shows a loading spinner:

import { ActivityIndicator } from 'react-native';

// Simple usage
<ActivityIndicator size="large" color="#3498db" />

// With loading state
{isLoading ? (
  <ActivityIndicator size="small" color="#999" />
) : (
  <Text>Data loaded</Text>
)}

Switch

Toggle switch component:

import { Switch } from 'react-native';

// Controlled component
<Switch
  value={isEnabled}
  onValueChange={setIsEnabled}
  trackColor={{ false: '#767577', true: '#81b0ff' }}
  thumbColor={isEnabled ? '#3498db' : '#f4f3f4'}
/>

Modal

Presents content above the current screen:

import { Modal, View, Text } from 'react-native';

<Modal
  visible={modalVisible}
  animationType="slide"
  transparent={true}
  onRequestClose={() => setModalVisible(false)}
>
  <View style={styles.modalContainer}>
    <View style={styles.modalContent}>
      <Text>Medication details</Text>
    </View>
  </View>
</Modal>

Alert

Show native alert dialogs:

import { Alert } from 'react-native';

// Simple alert
Alert.alert(
  "Medication Reminder",
  "Time to take your medication!",
  [
    { text: "Dismiss", style: "cancel" },
    { text: "Take Now", onPress: () => markAsTaken() }
  ]
);

Component Selection Guidelines

How to choose the right component for your UI needs:

For Layout

  • View: General containers and layouts
  • ScrollView: Scrollable content (moderate size)
  • FlatList/SectionList: Long lists of data
  • SafeAreaView: Content inside safe area (notch)

For User Input

  • TextInput: Text entry
  • Switch: Boolean toggles
  • Pressable/TouchableOpacity: Custom buttons
  • Button: Simple platform buttons

For Display

  • Text: Any textual content
  • Image: Photos and icons
  • ImageBackground: Background images

For Utility

  • ActivityIndicator: Loading states
  • Modal: Overlay content
  • Alert: System alerts
  • StatusBar: Customize status bar
Component Philosophy

Start with the simplest component that meets your needs, then add complexity as required. Compose components to create more complex UI patterns.

Best Practices

  • Component Composition: Create reusable UI components by composing core components
  • Proper Nesting: Always put text in Text components, not directly in View
  • Consistent Styling: Use a consistent styling approach across your app
  • Performance Awareness: Use FlatList for long lists instead of ScrollView
  • Accessibility: Add accessibility props like accessibilityLabel and accessibilityHint
  • Platform Adaptation: Test on both iOS and Android and adapt as needed
  • Error Handling: Add appropriate error handling for images and network requests
Practical Tip

Start by building a component library of common UI elements specific to your pharmaceutical app (pill displays, dosage indicators, etc.) using these core components as building blocks.
