Section 1

Development Tools Installation

Module 2 | 30-45 minutes

Learning Objectives

  • Install all required development tools for React Native development on macOS
  • Configure your system for both iOS and Android development
  • Understand the purpose of each tool in the React Native development workflow
  • Verify your installation with appropriate checks

Time: 30-45 minutes

Prerequisites: Basic command line familiarity

Essential Development Tools Overview

React Native development requires several tools and dependencies to create, build, and run applications across different platforms:

For iOS Development

  • Xcode & Command Line Tools
  • iOS Simulator
  • CocoaPods (dependency manager)
iOS Developers: Ensure Xcode is updated to the latest stable version with Command Line Tools installed.

For Android Development

  • Android Studio
  • Android SDK
  • JDK (Java Development Kit)
  • Android Virtual Device (Emulator)
Android Developers: Ensure your SDK is configured specifically for React Native development.

Common Tools for All Platforms

  • Node.js & npm
  • Watchman (file watching service)
  • Expo CLI & Expo Go app
  • Git (version control)
Web Developers: You're likely familiar with Node.js and npm, but you'll need additional tools for mobile development.

Installing Xcode and Command Line Tools

Xcode is Apple's integrated development environment (IDE) required for iOS development:

  1. Open the App Store and search for "Xcode"
  2. Click "Get" or "Download" to install Xcode (large download - may take time)
  3. Open Xcode and accept any license agreements
  4. Install Command Line Tools via Terminal:
xcode-select --install
Deep Dive: Command Line Tools

The Command Line Tools package gives you essential development tools like Git, the Apple LLVM compiler, and build tools necessary for native development. React Native uses these tools during the build process to compile native code for iOS.

Remind participants that Xcode is several GB in size and may require significant download time. If participants are attending a workshop, suggest downloading Xcode in advance.

Installing Homebrew, Node.js, and Watchman

Install Homebrew

Package manager for macOS that simplifies tool installation:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Follow on-screen instructions to complete setup.

Install Node.js

JavaScript runtime that powers React Native:

brew install node

Verify installation:

node --version  # Should show v16.x or later
npm --version   # Should show v8.x or later

Install Watchman

File watching service that improves performance:

brew install watchman
Deep Dive: Watchman's Role

Watchman optimizes the development workflow by efficiently observing file changes. When you modify your code, Watchman notifies the Metro bundler, which then rebundles your JavaScript. This enables features like Fast Refresh.

Setting Up Android Development Environment

Install JDK

brew install --cask adoptopenjdk/openjdk/adoptopenjdk11

Install Android Studio

  1. Download from developer.android.com/studio
  2. Follow installation wizard
  3. Select: Android SDK, Platform, Virtual Device

Configure Android SDK

  1. In Android Studio, go to "More Actions" > "SDK Manager"
  2. In SDK Platforms tab, select:
    • Android 13 (Tiramisu)
    • Android 12 (S)
    • Android 11 (R)
  3. In SDK Tools tab, select:
    • Android SDK Build-Tools
    • Android SDK Command-line Tools
    • Android Emulator
    • Android SDK Platform-Tools
Environment Variables

Add these to your shell profile file (`.zshrc`, `.bash_profile`):

export ANDROID_HOME=$HOME/Library/Android/sdk
export PATH=$PATH:$ANDROID_HOME/emulator
export PATH=$PATH:$ANDROID_HOME/tools
export PATH=$PATH:$ANDROID_HOME/tools/bin
export PATH=$PATH:$ANDROID_HOME/platform-tools

Installing Expo CLI and Mobile Testing Apps

Install Expo CLI

Command-line utility for creating and managing Expo projects:

npm install -g expo-cli

Verify installation:

expo --version

Install Expo Go on Mobile Devices

For testing on physical devices:

  • iOS: Download Expo Go from App Store
  • Android: Download Expo Go from Google Play

This allows you to test your applications without building and deploying them to a device manually.

What is Expo?

Expo is a framework and platform built around React Native that provides a set of tools and services for developing, building, and deploying React Native applications. It simplifies the development process, especially for beginners.

Verifying Your Installation

Verify Node.js and npm

node --version
npm --version

Verify Watchman

watchman --version

Verify Xcode Command Line Tools

xcode-select --print-path

Should output: /Applications/Xcode.app/Contents/Developer

Verify Android SDK

sdkmanager --list | grep "installed"

Should list installed Android SDK packages

Verify Expo CLI

expo --version

Verify Environment Variables

echo $ANDROID_HOME

Should point to your Android SDK location

Troubleshooting Common Installation Issues

Xcode Command Line Tools Issues

sudo xcode-select --reset
xcode-select --install

Node.js Version Conflicts

Use nvm to manage multiple Node.js versions:

brew install nvm

# Then follow instructions to configure nvm
nvm install 16
nvm use 16

Android SDK Configuration Issues

  1. Check that the path in ANDROID_HOME exists
  2. Make sure you've sourced your profile file:
    3. source ~/.zshrc  # Or appropriate file
  3. Verify with:
    4. echo $ANDROID_HOME
Deep Dive: Android Environment Variables

React Native's build process uses these environment variables to locate the Android SDK tools it needs for compiling and packaging your application. Without them, the Metro bundler and React Native CLI won't be able to communicate with the Android tools.
