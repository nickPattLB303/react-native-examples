Introduction to Expo Projects

Expo provides a streamlined way to create, develop, and deploy React Native applications:

  • Simplified project setup and configuration
  • Pre-configured native modules for common features
  • Cross-platform development with minimal setup
  • Easy deployment and updates

For Web Developers

Expo projects are similar to Create React App for web development, but with additional mobile-specific tools and configurations.

For iOS/Android Developers

Expo abstracts away much of the native configuration you're used to, allowing you to focus on the application logic rather than build configurations.

Creating a New Expo Project

Using create-expo-app

The recommended way to create a new Expo project:

npx create-expo-app@latest MyPharmacyApp

This command:

  1. Downloads and executes the latest version of create-expo-app
  2. Creates a new directory called MyPharmacyApp
  3. Initializes a React Native project with Expo configuration
  4. Installs all necessary dependencies
Deep Dive: npx

The npx command is part of npm and allows you to run Node.js packages without installing them globally. When you run npx create-expo-app, it downloads the latest version of the package, runs it, and then removes it from your system.

Exploring Available Templates

Expo provides several templates as starting points for your project:

npx create-expo-app@latest --template

Common Templates

  • blank: A minimal app with just the essentials
  • blank (TypeScript): Same as blank but with TypeScript
  • tabs: A tab-based navigation template
  • navigation: A starter with React Navigation pre-configured

Using a Specific Template

npx create-expo-app@latest MyPharmacyApp --template blank

For this course, we'll use the default template, which is a good starting point for most applications.

Tip

For more complex applications, consider using a template with navigation already set up to save time on initial configuration.

Understanding Project Structure

MyPharmacyApp/
├── .expo/                  # Expo configuration files
├── .expo-shared/           # Expo shared configuration
├── assets/                 # Static assets like images, fonts
├── node_modules/           # External dependencies
├── app/                    # Main application code
│   ├── index.tsx           # Entry point of your app
│   ├── _layout.tsx         # Layout component
├── .gitignore              # Git ignore file
├── app.json                # Expo configuration
├── babel.config.js         # Babel configuration
├── package.json            # Project metadata and deps
├── tsconfig.json           # TypeScript configuration
└── README.md               # Project documentation

Key Directories

  • app/: Contains your application code
  • assets/: Contains static resources (images, fonts)
  • node_modules/: Contains dependencies

Key Files

  • app.json: Expo configuration
  • package.json: Project metadata and scripts
  • app/index.tsx: Main component/entry point

Key Project Files

app/index.tsx

The entry point of your application:

import { StyleSheet, Text, View } from 'react-native';

export default function Index() {
  return (
    
      Hello world!
    
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#fff',
    alignItems: 'center',
    justifyContent: 'center',
  },
});
Deep Dive: Components

React Native uses a component-based architecture similar to React for web, but with native components instead of DOM elements. A View renders as a UIView on iOS and as an android.view on Android.

assets/ Directory

Contains static resources used by your app:

  • Images
  • Fonts
  • Audio files
  • Other static resources

Configuration Files

app.json

Contains configuration for your Expo project:

{
  "expo": {
    "name": "MyPharmacyApp",
    "slug": "MyPharmacyApp",
    "version": "1.0.0",
    "orientation": "portrait",
    "icon": "./assets/icon.png",
    "splash": {
      "image": "./assets/splash.png",
      "resizeMode": "contain",
      "backgroundColor": "#ffffff"
    },
    "ios": {
      "supportsTablet": true
    },
    "android": {
      "adaptiveIcon": {
        "foregroundImage": "./assets/adaptive-icon.png",
        "backgroundColor": "#ffffff"
      }
    }
  }
}

package.json

Contains metadata and scripts for your project:

{
  "name": "MyPharmacyApp",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "start": "expo start",
    "android": "expo start --android",
    "ios": "expo start --ios",
    "web": "expo start --web"
  },
  "dependencies": {
    "expo": "~49.0.13",
    "expo-status-bar": "~1.6.0",
    "react": "18.2.0",
    "react-native": "0.72.5"
    // ...
  }
}
Deep Dive: app.json

The app.json file is used by Expo's build service to configure your app for different platforms. When you build your app for iOS, Android, or web, Expo reads this file to generate the appropriate native configuration files.

Using Expo CLI Commands

Starting Your Project

npx expo start

This starts the Metro bundler and displays a QR code for the Expo Go app.

Platform-Specific Commands

npx expo start --ios     # iOS simulator
npx expo start --android # Android emulator
npx expo start --web     # Web browser

Additional Useful Commands

npx expo doctor          # Check for issues
npx expo upgrade         # Upgrade Expo SDK
npx expo install         # Install packages
Tip

Always use npx expo install instead of npm install when adding packages to your Expo project. This ensures you get versions compatible with your Expo SDK version.

Deep Dive: Metro Bundler

When you run npx expo start, the Metro bundler creates a development server that watches your files for changes, bundles your JavaScript code, serves the bundled code to your device or simulator, and enables features like Hot Reloading and debugging.
