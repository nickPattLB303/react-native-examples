What is React?

React is a JavaScript library for building user interfaces, primarily focused on the view layer of applications.

  • Developed and maintained by Facebook (now Meta)
  • First released in 2013
  • Used by thousands of companies including Instagram, Netflix, Airbnb
  • Designed to make UI development more predictable and efficient
  • Focused on component-based architecture
Key Insight

React is not a full-featured framework like Angular. It's a library that focuses specifically on the UI layer, allowing you to integrate it with other libraries for routing, state management, etc.

Make sure to emphasize that React is focused on the UI layer, which is why it's often combined with other libraries. This "pick what you need" philosophy is quite different from more opinionated frameworks like Angular.

React's Core Philosophy

React's design is guided by several key principles:

Component-Based

  • UI broken down into reusable components
  • Each component encapsulates its own logic, state, and appearance
  • Components can be composed to build complex UIs

Declarative

  • Describe what the UI should look like
  • React handles DOM updates efficiently
  • Makes code more predictable and easier to debug
Conceptual Model

Think of React as a function where your state is the input and the UI is the output: UI = F(state)

Component-Based Architecture

In React, components are the building blocks of your application:

  • Reusable: Components can be used multiple times across an application
  • Composable: Smaller components can be combined to create more complex UIs
  • Maintainable: Changes to one component don't affect others (when properly designed)
  • Testable: Components can be tested in isolation
  • Encapsulated: Components manage their own state and behavior
For Android Developers

Components are conceptually similar to custom Views in Android, but with a more formalized composition model.

For iOS Developers

Components are similar to UIViews or SwiftUI views, but with more emphasis on composition over inheritance.

Declarative Programming

Imperative (Traditional DOM)

Explicitly tell the DOM what to do at each step


// Get references
const button = document.getElementById('btn');
const counter = document.getElementById('counter');
let count = 0;

// Add event listener
button.addEventListener('click', () => {
  count++;
  counter.textContent = `Count: ${count}`;
});

Describe the UI for any given state

Key Difference

With imperative code, you describe how to update the UI. With declarative code, you describe what the UI should look like based on state, and React handles the updates.

Unidirectional Data Flow

Data in React flows in one direction, making applications more predictable:

Parent Component
↓ props
Child Component
↑ callbacks
Parent Component (state updates)
  • Data flows down from parent components to children via props
  • Child components cannot directly modify parent data
  • Child components communicate up via callbacks provided by parents
  • This pattern creates a predictable data flow that's easier to debug
Mental Model

Think of props like function parameters. They're passed in, but the function can't modify the original values directly.

The Evolution of React

2011

Initial Development

Developed internally at Facebook by Jordan Walke

2013

Public Release

React open-sourced at JSConf US

2015

React Native

React model extended to mobile platforms

2016-2018

Fiber Rewrite & Context API

Complete internal rewrite for better performance

2019

Hooks

Introduced hooks for state and lifecycle in functional components

2022+

Concurrent Features

Improved rendering with concurrent mode and automatic batching

The hooks introduction in 2019 was a pivotal moment that changed how we write React code, moving away from class components to functional components with hooks for state and lifecycle.

React vs. React Native

React and React Native share core principles but differ in implementation:

Feature React (Web) React Native
Rendering Target DOM (browsers) Native UI components
Base Components div, span, p, etc. View, Text, Image, etc.
Styling CSS, CSS-in-JS JavaScript objects with CSS-like properties
Events Browser events (onClick, etc.) Touch events (onPress, etc.)
Animation CSS transitions, Web Animations API Animated API
Navigation React Router, etc. React Navigation, etc.
The Common Core

Components, props, state, and lifecycle are consistent across both platforms. If you know React, you already know 80% of React Native.

React in Modern Application Development

React has influenced the broader JavaScript ecosystem:

React Ecosystem

  • Meta Frameworks: Next.js, Gatsby, Remix
  • State Management: Redux, MobX, Recoil
  • UI Libraries: Material UI, Chakra UI
  • Testing: React Testing Library, Jest

Influence on Development

  • Component-based architecture
  • Virtual DOM concepts
  • Declarative UI patterns
  • Composition over inheritance
  • Functional programming principles
Cross-Platform Impact

React's component model has been adapted for native mobile (React Native), desktop (Electron), VR (React 360), and even hardware interfaces.

Why Learn React Before React Native

Understanding React fundamentals provides the foundation for effective React Native development:

  1. Same Core Concepts: Components, props, state, and hooks are identical
  2. Simpler Debugging: Web React is easier to debug than React Native
  3. Focus on Concepts: Learn principles without mobile-specific complexities
  4. Larger Community: More resources and solutions available for React
  5. Transferable Skills: Skills apply directly to React Native development
For Web Developers

Your existing React knowledge is a huge advantage! Focus on understanding the different primitive components and mobile-specific patterns.

For Mobile Developers

Learning React fundamentals first will help you understand why React Native works the way it does, making the transition much smoother.

Summary: Introduction to React

  • React is a JavaScript library for building user interfaces with components
  • Key principles: component-based architecture, declarative programming, and unidirectional data flow
  • React has evolved from a web library to powering multiple platforms
  • React and React Native share core concepts but have platform-specific implementations
  • Understanding React fundamentals provides a strong foundation for React Native development
